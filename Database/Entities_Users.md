# شرح Entities - Users Module

## 1. Users (اليوزر الأساسي)

**ده إيه؟**
- ده الـ entity الأساسي لأي حد هيستخدم النظام
- بيرث من `IdentityUser` عشان يستخدم نظام الـ Authentication بتاع ASP.NET

**الـ Properties:**

### Id (Guid)
- **ده إيه:** رقم تعريف اليوزر (Primary Key)
- **استخدامه:** عشان نعرف كل يوزر بشكل unique
- **مثال:** `3fa85f64-5717-4562-b3fc-2c963f66afa6`

### UserName (string)
- **ده إيه:** اسم المستخدم اللي هيدخل بيه
- **استخدامه:** للـ Login
- **مثال:** `ahmed.hassan` أو `ahmed@example.com`

### Email (string)
- **ده إيه:** الإيميل بتاع اليوزر
- **استخدامه:** للتواصل والـ Login
- **مثال:** `ahmed@example.com`
- **ملاحظة:** لازم يكون Unique (مفيش 2 يوزر بنفس الإيميل)

### PasswordHash (string)
- **ده إيه:** الباسورد بعد ما يتشفر
- **استخدامه:** عشان نحفظ الباسورد بشكل آمن
- **ملاحظة:** مش بنحفظ الباسورد نفسه، بنحفظه مشفر

### FirstName (string)
- **ده إيه:** الاسم الأول
- **استخدامه:** عشان نعرض اسم اليوزر
- **مثال:** `أحمد`

### LastName (string)
- **ده إيه:** اسم العائلة
- **استخدامه:** عشان نعرض الاسم الكامل
- **مثال:** `حسن`

### DateOfBirth (DateTime?)
- **ده إيه:** تاريخ الميلاد
- **استخدامه:** عشان نعرف سن اليوزر
- **مثال:** `1995-05-15`
- **ملاحظة:** مش إجباري (?)

### Gender (Gender?)
- **ده إيه:** النوع (ذكر/أنثى)
- **استخدامه:** بيانات شخصية
- **القيم:** `Male = 1` أو `Female = 2`
- **ملاحظة:** مش إجباري (?)

### ProfilePictureUrl (string?)
- **ده إيه:** رابط صورة البروفايل
- **استخدامه:** عشان نعرض صورة اليوزر
- **مثال:** `https://storage.com/profiles/ahmed.jpg`
- **ملاحظة:** مش إجباري (?)

### UserType (UserType)
- **ده إيه:** نوع اليوزر (طالب ولا مرشد)
- **استخدامه:** عشان نعرف اليوزر ده Student ولا Mentor
- **القيم:** `Student = 1` أو `Mentor = 2`
- **ملاحظة:** إجباري

### CreatedAt (DateTime)
- **ده إيه:** تاريخ إنشاء الحساب
- **استخدامه:** عشان نعرف اليوزر اتسجل إمتى
- **مثال:** `2025-01-15 10:30:00`

### IsDeleted (bool)
- **ده إيه:** هل الحساب متمسوح ولا لأ
- **استخدامه:** Soft Delete (مش بنمسح من الداتابيز، بنعلم عليه بس)
- **القيم:** `true` أو `false`

### DeletedAt (DateTime?)
- **ده إيه:** تاريخ مسح الحساب
- **استخدامه:** عشان نعرف الحساب اتمسح إمتى
- **ملاحظة:** بيبقى null لو الحساب مش متمسوح

## 2. StudentProfile (بروفايل الطالب)

**ده إيه؟**
- ده البيانات الإضافية الخاصة بالطالب
- كل طالب ليه Users + StudentProfile

**الـ Properties:**

### Id (int)
- **ده إيه:** رقم تعريف البروفايل (Primary Key)
- **استخدامه:** عشان نعرف كل بروفايل بشكل unique

### UserId (Guid)
- **ده إيه:** رقم اليوزر اللي البروفايل ده تابع له (Foreign Key)
- **استخدامه:** عشان نربط البروفايل باليوزر
- **العلاقة:** One-to-One مع Users

### Education (string?)
- **ده إيه:** المستوى التعليمي أو التخصص
- **استخدامه:** عشان نعرف الطالب بيدرس إيه
- **مثال:** `Computer Science - Cairo University`
- **ملاحظة:** مش إجباري

### EmploymentStatus (EmploymentStatus?)
- **ده إيه:** حالة التوظيف
- **استخدامه:** عشان نعرف الطالب بيشتغل ولا لأ
- **القيم:** 
  - `Student = 1` (طالب فقط)
  - `Employed = 2` (بيشتغل)
  - `Unemployed = 3` (مش بيشتغل)
  - `Freelancer = 4` (فريلانسر)
- **ملاحظة:** مش إجباري

### YearsOfExperience (int?)
- **ده إيه:** سنين الخبرة
- **استخدامه:** عشان نعرف الطالب عنده خبرة قد إيه
- **مثال:** `2` (سنتين خبرة)
- **ملاحظة:** مش إجباري

### Experience (string?)
- **ده إيه:** تفاصيل الخبرة
- **استخدامه:** عشان الطالب يكتب خبراته بالتفصيل
- **مثال:** `Worked as Junior Developer at XYZ Company`
- **ملاحظة:** مش إجباري

### CvUrl (string?)
- **ده إيه:** رابط الـ CV
- **استخدامه:** عشان الطالب يرفع الـ CV بتاعه
- **مثال:** `https://storage.com/cvs/ahmed_cv.pdf`
- **ملاحظة:** مش إجباري

### DailyStudyHours (decimal?)
- **ده إيه:** عدد ساعات الدراسة اليومية
- **استخدامه:** عشان نعرف الطالب بيقدر يدرس كام ساعة في اليوم
- **مثال:** `3.5` (3 ساعات ونص)
- **ملاحظة:** مش إجباري

### CurrentSkills (string?)
- **ده إيه:** المهارات الحالية (JSON Array)
- **استخدامه:** عشان نعرف الطالب بيعرف إيه دلوقتي
- **مثال:** `["C#", "JavaScript", "SQL"]`
- **ملاحظة:** مش إجباري، بنحفظه كـ JSON

### PreferredFields (string?)
- **ده إيه:** المجالات المفضلة (JSON Array)
- **استخدامه:** عشان نعرف الطالب عايز يتعلم إيه
- **مثال:** `["Backend Development", "DevOps"]`
- **ملاحظة:** مش إجباري، بنحفظه كـ JSON

### PreferredLearningStyle (LearningStyle?)
- **ده إيه:** أسلوب التعلم المفضل
- **استخدامه:** عشان نعرف الطالب بيحب يتعلم إزاي
- **القيم:**
  - `Visual = 1` (بصري - فيديوهات)
  - `Auditory = 2` (سمعي - بودكاست)
  - `Reading = 3` (قراءة - مقالات)
  - `Kinesthetic = 4` (عملي - مشاريع)
- **ملاحظة:** مش إجباري

### CreatedAt, IsDeleted, DeletedAt
- نفس الشرح اللي في Users

## 3. MentorProfile (بروفايل المرشد)

**ده إيه؟**
- ده البيانات الإضافية الخاصة بالمرشد
- كل مرشد ليه Users + MentorProfile

**الـ Properties:**

### Id (int)
- **ده إيه:** رقم تعريف البروفايل (Primary Key)

### UserId (Guid)
- **ده إيه:** رقم اليوزر اللي البروفايل ده تابع له (Foreign Key)
- **العلاقة:** One-to-One مع Users

### JobTitle (string)
- **ده إيه:** المسمى الوظيفي
- **استخدامه:** عشان نعرف المرشد بيشتغل إيه
- **مثال:** `Senior Backend Developer`
- **ملاحظة:** إجباري

### YearsOfExperience (int)
- **ده إيه:** سنين الخبرة
- **استخدامه:** عشان نعرف المرشد عنده خبرة قد إيه
- **مثال:** `8` (8 سنين خبرة)
- **ملاحظة:** إجباري

### ProfessionalBio (string?)
- **ده إيه:** نبذة مهنية عن المرشد
- **استخدامه:** عشان المرشد يعرّف نفسه
- **مثال:** `Experienced developer with 8+ years in backend development`
- **ملاحظة:** مش إجباري

### AvailabilitySchedule (string?)
- **ده إيه:** جدول المواعيد المتاحة (JSON)
- **استخدامه:** عشان نعرف المرشد متاح إمتى
- **مثال:** `{"Sunday": ["10:00-12:00", "14:00-16:00"]}`
- **ملاحظة:** مش إجباري، بنحفظه كـ JSON

### PricePerHour (decimal?)
- **ده إيه:** السعر بالساعة
- **استخدامه:** لو المرشد بيقدم خدمات مدفوعة
- **مثال:** `50.00` (50 جنيه/دولار في الساعة)
- **ملاحظة:** مش إجباري

### SpokenLanguages (string?)
- **ده إيه:** اللغات اللي بيتكلمها (JSON Array)
- **استخدامه:** عشان نعرف المرشد بيتكلم إيه
- **مثال:** `["Arabic", "English"]`
- **ملاحظة:** مش إجباري، بنحفظه كـ JSON

### AverageRating (decimal)
- **ده إيه:** متوسط التقييم
- **استخدامه:** عشان نعرف المرشد تقييمه كام
- **مثال:** `4.5` (من 5)
- **ملاحظة:** بيتحسب تلقائياً من تقييمات الطلاب

### TotalReviews (int)
- **ده إيه:** عدد التقييمات
- **استخدامه:** عشان نعرف كام طالب قيّم المرشد
- **مثال:** `25` (25 تقييم)
- **ملاحظة:** بيتحسب تلقائياً

### CreatedAt, IsDeleted, DeletedAt
- نفس الشرح اللي في Users

## ملخص العلاقات

```
Users (اليوزر الأساسي)
  ├─ 1:1 → StudentProfile (لو كان طالب)
  └─ 1:1 → MentorProfile (لو كان مرشد)
```

**مثال:**
```
User: Ahmed
  ├─ UserType = Student
  └─ StudentProfile:
      ├─ Education = "Computer Science"
      └─ DailyStudyHours = 3

User: Sara
  ├─ UserType = Mentor
  └─ MentorProfile:
      ├─ JobTitle = "Senior Developer"
      └─ YearsOfExperience = 8
```
